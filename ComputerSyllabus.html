Chapter 2: Number System, Conversion and Boolean Logic

Introduction

A number system is a writing system for denoting numbers logically using digits or other symbols. It provides a unique representation of every number and reflects the arithmetic and algebraic structure of numbers. In computing, several number systems are utilized:

  • Decimal (Base-10): The standard system for denoting integer and non-integer numbers, using digits from 0 to 9.
  • Binary (Base-2): Utilized internally by almost all modern computers and computer-based devices because it is straightforward to implement with digital electronic circuitry. It uses only two digits: 0 and 1.
  • Octal (Base-8): Uses digits from 0 to 7. Each octal digit represents three binary digits, making it more compact for representing binary numbers.
  • Hexadecimal (Base-16): Uses digits from 0 to 9 and letters A to F. Each hexadecimal digit represents four binary digits, providing a more human-friendly representation of binary-coded values.

Understanding these systems is crucial for tasks such as data representation, memory addressing, and low-level programming.

Boolean Algebra

Boolean algebra is a branch of mathematics that deals with variables that have two distinct values: true and false, typically denoted as 1 and 0, respectively. It forms the foundation of digital logic and is essential for designing and understanding computer circuits and algorithms. Key concepts include:

  • Logical Operations: AND, OR, and NOT operations that define how binary variables interact.
  • Truth Tables: Tables that list all possible values of logical variables and the result of applying logical operations to them.
  • Simplification of Expressions: Techniques like De Morgan's laws and Karnaugh maps are used to simplify complex Boolean expressions, optimizing digital circuits.

Proficiency in Boolean algebra is vital for tasks such as circuit design, algorithm optimization, and understanding the underlying principles of computer operations.

Number System Conversions

1. Binary to Decimal Conversion

To convert a binary number (base 2) to decimal (base 10), you multiply each digit by the power of 2 and sum the results.

Steps:

  1. Write the binary number.
  2. Starting from the right, assign each digit a power of 2 (2^0, 2^1, 2^2, etc.).
  3. Multiply each binary digit by the corresponding power of 2.
  4. Sum the results.

Example: Convert 1011 (binary) to decimal.

Start from the right:

  • 1 = 2^3 = 8
  • 0 = 2^2 = 0
  • 1 = 2^1 = 2
  • 1 = 2^0 = 1

Now sum the results: 8 + 0 + 2 + 1 = 11 (decimal)

So, 1011 (binary) = 11 (decimal).

2. Decimal to Binary Conversion

To convert a decimal number (base 10) to binary (base 2), divide the number by 2, recording the remainder each time. Then, read the remainders from bottom to top.

Steps:

  1. Divide the decimal number by 2.
  2. Write down the remainder.
  3. Continue dividing by 2 until the quotient is 0.
  4. The binary number is the string of remainders read from bottom to top.

3. Binary to Octal Conversion

To convert binary to octal (base 8), group the binary digits in sets of three, starting from the right. If there are not enough digits to make a group of three, add leading zeros. Then, convert each group of three digits to its octal equivalent.

Steps:

  1. Group the binary number into sets of three digits, starting from the right.
  2. Convert each group to octal.

Example: Convert 110101 (binary) to octal.

Group the digits: 110 101

Convert each group to octal:

  • 110 (binary) = 6 (octal)
  • 101 (binary) = 5 (octal)

So, 110101 (binary) = 65 (octal).

4. Octal to Binary Conversion

To convert octal to binary, convert each octal digit to its 3-bit binary equivalent.

Steps:

  1. Write each octal digit in its 3-bit binary equivalent.

Example: Convert 35 (octal) to binary.

Convert each octal digit to binary:

  • 3 (octal) = 011 (binary)
  • 5 (octal) = 101 (binary)

So, 35 (octal) = 011101 (binary).

5. Binary to Hexadecimal Conversion

To convert binary to hexadecimal (base 16), group the binary digits in sets of four, starting from the right. If there are not enough digits to make a group of four, add leading zeros. Then, convert each group of four binary digits to its hexadecimal equivalent.

Steps:

  1. Group the binary number into sets of four digits, starting from the right.
  2. Convert each group to hexadecimal.

Example: Convert 1011101 (binary) to hexadecimal.

Group the digits: 1011 101

Add leading zeros to make a full group of four: 1011 1101

Convert each group to hexadecimal:

  • 1011 (binary) = B (hexadecimal)
  • 1101 (binary) = D (hexadecimal)

So, 1011101 (binary) = BD (hexadecimal).

6. Hexadecimal to Binary Conversion

To convert hexadecimal to binary, convert each hexadecimal digit to its 4-bit binary equivalent.

Steps:

  1. Write each hexadecimal digit in its 4-bit binary equivalent.

Example: Convert A3 (hexadecimal) to binary.

Convert each hexadecimal digit to binary:

  • A (hexadecimal) = 1010 (binary)
  • 3 (hexadecimal) = 0011 (binary)

So, A3 (hexadecimal) = 10100011 (binary).

Chapter 3: Computer Softeware Operating System

Introduction

An operating system (OS) is an integrated set of programs that control overall resources such as CPU, memory, and input-output devices of the computer system. The major objective of an operating system is to improve the performance and efficiency of a computer system. Like the manager of a company, an operating system is responsible for the smooth and efficient functioning of the entire computer system. The operating system provides the platform for other application programs/software to run and execute. It provides the user with an interface so that the user can easily communicate with computers, which is more convenient to use and operate.

Functions of Operating System

  • Controls, monitors, and coordinates the overall operation of our computer system.
  • Acts as an interface between the user and the computer.
  • Provides a platform to develop, run, and execute other computer programs.
  • Manages hardware resources such as CPU, memory, input-output terminals, networking equipment, etc.
  • Hides programming and hardware complexity from the user.

Types of Operating System

A) Types of Operating Systems Based on Processing

  • Multi-programming OS: Executes multiple user programs on a single processor, improving resource utilization. Examples: multi-tasking, multi-processing, and multi-user systems.
  • Multi-tasking OS: Allows multiple tasks to run concurrently in a single-user environment. Examples: MS-Windows, Linux, macOS.
  • Multiprocessing OS: Uses more than one processor to run programs simultaneously, improving execution speed. Example: UNIX.
  • Time-Sharing OS: Allows multiple users to share CPU time. Tasks switch frequently, providing an immediate response. Examples: MS-DOS, UNIX.
  • Batch Processing OS: Groups similar jobs into batches for sequential processing without user interaction during execution. Common in payroll, weather forecasting, etc.
  • Multi-threading OS: Divides a program into threads that can be executed concurrently, improving efficiency.
  • Real-time OS: Processes tasks in a predefined time, suitable for critical operations like flight seat availability and rocket launching systems.
  • Online Processing OS: Processes transactions immediately as they occur, allowing user interaction during processing.

B) Types of Operating Systems Based on User Interface (UI)

  • Character User Interface (CUI): Operates using text-based commands. Less user-friendly, requiring users to memorize commands. Examples: MS-DOS, UNIX.
  • Graphical User Interface (GUI): Allows interaction using graphics, icons, and dialogue boxes. More user-friendly and easier for non-technical users.

Software

Software is the intangible set of instructions, data, or programs that enable computers and other devices to perform specific tasks. It encompasses a wide range of applications, from operating systems that manage hardware resources to specialized programs designed for tasks like word processing, image editing, or web browsing. Software development involves a systematic process of designing, coding, testing, and deploying these programs, often requiring collaboration among developers, designers, and testers. The quality and efficiency of software are crucial for its usability and reliability, impacting everything from personal productivity to complex industrial operations. Updates and maintenance are essential for addressing bugs, enhancing features, and ensuring compatibility with evolving hardware and software environments.

Chapter 5: Programming Concepts and Logics

5.1 Introduction

As we know, every computerized device does not work by itself. They need to be given some sort of command that makes the computer work. These commands help to instruct the computer and program specific actions required by the user. Since the commands are given in the form of computer language, we need programming languages to write the set of instructions that command the computer to perform certain actions, collectively called programs. These programs are again integrated in order to make a complete software. Simply put, software is the collection of different interrelated programs that perform specific functions.

Programs written in any type of programming language are not understood by the computer; hence we need some sort of language translator or processor, such as an assembler, compiler, or interpreter, which has the basic function of converting programs written in any type of language into machine-level language.

The program written by the programmer is known as the source program. After converting, it becomes a program.

5.2 Qualities of Good Programs

  • It should be easily understood.
  • A program should be correct; it should be error-free.
  • It should be reliable.
  • It should have an easily understandable user interface.
  • It should be portable and flexible.

5.3 Programming Language

In order to make communication between the user and computer, we need a computer language that helps the user generate commands to perform as per the requirement. This language with which we can give instructions to the computer is known as a programming language. Programming languages are sets of different keywords, variables, operators, loops, and other entities using different character sets, including numbers, special symbols, and alphanumeric values.

Hence, the process of writing the programming language is known as programming, and the person who writes programs is a programmer.

5.4 Types of Programming Languages

There are several types of programming languages that can be categorized as follows:

Low Level Language

Low-level languages are machine-dependent languages, meaning programs written for one type of system cannot be run on another system. Developers should have detailed analysis and knowledge about the system for which they are going to write a program. Hence, programming in low-level language is very much difficult and time-consuming.

Different types of low-level programming languages are:

Machine Level Language (1GL)

This language consists of a sequence of 0's and 1's to generate instructions. Since it uses binary numbers, this type of language is directly understood by the processor. So, it has higher execution speed. It is also a machine-dependent language, in which programmers should have detailed knowledge of the system.

Advantages:

  • It is directly understood by the processor, so execution speed is relatively high.
  • Language translator or processor is not needed.
  • It can be used to program specific-purpose computers.

Disadvantages:

  • It is difficult and time-consuming to develop and debug programs.
  • It is a machine-dependent language. So, programs developed for one system cannot be operated on another system.
  • All the syntax and commands are in the form of binary numbers, which are difficult to remember.
  • Programmers should have detailed knowledge about the particular system and its architecture.

Assembly Language (2GL)

Assembly language is also an example of a low-level language. In this language, instead of writing instructions in a series of 0's and 1's, we can use mnemonics (symbolic instructions) like ADD, SUB, RST, DIV, MOD, and so on. Since it is closer to machine-level language, a programmer should have detailed knowledge about the computer's internal architecture. This language is faster in comparison to high-level languages. Since this language is not directly understood by the computer, we need a language translator like an assembler to convert it into machine-level language.

Advantages:

  • It is easier to write, debug, and understand programs written in assembly-level language compared to machine-level language.
  • Program execution is faster compared to high-level languages.
  • Since they are machine-dependent, they are used to develop different device drivers.

Disadvantages:

  • It is a machine-dependent language, i.e., a program made for one processor doesn't run on another processor.
  • The use of mnemonics code makes assembly language much more complex.
  • Program development and debugging is more difficult and time-consuming compared to high-level languages.

High Level Language

This language is closer to the English language. High-level language code is written in an English-like structure using mathematical notation. Since it is similar to the English language, it is easier to develop and debug programs. It is a machine-independent language (i.e., a program developed for one processor can work on another processor). Since high-level languages are not directly understood by computers, we need a language processor or translator such as a compiler and interpreter to convert programs written in high-level languages into machine-level language. FORTAN (Formula Translator), introduced in 1956, was the first high-level language. Nowadays, many high-level languages like C, C++, Python, JavaScript, etc., are used.

Advantages:

  • Since it is closer to the English language, programs written in this language are easier to write, debug, and understand.
  • Since it is machine-independent, a program written for one processor can work on another processor.
  • Programmers don't have to remember a large number of mnemonics and other unusual codes.
  • Program development is faster and requires less effort than other languages.

Disadvantages:

  • Computers don't understand high-level languages directly. So the program needs conversion before execution.
  • Program execution is slower compared to low-level languages.

Types of High-Level Languages

Procedural Oriented Language (3GL)

This type of language is a high-level language that primarily focuses on procedures rather than data. Hence, they are used to express the logic and the procedure of the program. Since it focuses only on procedure, it is complex and time-consuming to write a large program. This type of language follows a top-to-bottom approach, i.e., the main function is written at the bottom of the program. This type of language doesn't pose important and powerful features like data encapsulation, data inheritance, data extraction, and so on. Therefore, it has less security compared to object-oriented languages. Examples: C, FORTRAN, QBasic, etc.

Advantages:

  • Program development and debugging are easier compared to low-level languages.
  • More advanced and user-friendly software can be developed.
  • It is also a machine-independent language.
  • It is used as a general-purpose programming language.

Disadvantages:

  • A language translator or processor is required to execute the program.
  • Program execution is slower.
  • Data security is less in comparison to other high-level languages.
Problem/Object-Oriented Language (4GL)

This is the advanced form of high-level language which primarily focuses on data rather than procedure. It allows the user to specify what the output should be without describing all the details (i.e., procedure) of how the data should be manipulated. This type of language follows a bottom-up approach. That means all main functions are written at the bottom of the program, whereas classes and objects are described at the top of the program. Since it has several powerful features such as data encapsulation, data extraction, and inheritance, the data is more secure compared to procedural languages. Examples: C#, C++, Java, etc.

Advantages:

  • Web-based applications and software can be developed.
  • More advanced and user-friendly software can be developed.
  • It is also a machine-independent language.

Disadvantages:

  • A language translator is required to execute the program.
  • Program execution is slower.
  • It is difficult to develop hardware-oriented languages.
Natural Language (5GL)

Natural language uses simple statements of common communication language, where we could write statements that would look like normal sentences. It is still in the developing stage; computer scientists are working hard to develop such languages. However, programming languages like PROLOG (programming logic) are currently in use. For example, instead of writing some unusual code, programmers would write: "Who are the salesmen who have sold more than 30,000 products last month?"

Advantages:

  • It will be even easier to develop and debug programs.
  • It will also be machine-independent language.
  • More advanced and user-friendly programs will be made.

Disadvantages:

  • A language translator is required to execute the program.
  • Program execution is slower.
  • It is difficult to develop hardware-oriented languages.

5.5 Language Translator/Processor

A language translator is system development software that helps convert programs written in assembly or high-level languages (source programs) into machine-level languages (object programs). Since it is difficult and inconvenient to write a program in machine-level language, developers use several assembly and high-level languages that are not directly understood by the computer. Hence, different types of language processors are used to convert and make machine understandable. There are different types of language translators:

Types of Language Translators

Assembler

An assembler is a language processor that converts a program written in assembly-level language (source program) into machine-understandable language (object language). Since assembly-level language is closer to machine-level language, the conversion process is relatively quicker. It converts the program at once into machine-level language.

Compiler

A compiler is a language processor that helps convert a program written in high-level language (source program) into machine-level program (object program). It converts the whole program into machine-level language at once. It is the largest method of translating a program, but debugging is complex and time-consuming. Programming languages like C, C++, Java, etc., use compilers.

Interpreter

An interpreter is a language processor that converts a program written in high-level language (source program) into machine-level language (object program). It converts one statement at a time, so debugging is easier and less time-consuming. However, its program execution is slower than that of a compiler. Most new programming languages use interpreters, as they allocate less memory space. Programs like BASIC, C#, PHP, etc., use interpreters for conversion.

Difference Between Compiler and Interpreter

S.N. Compiler Interpreter
1 It translates the whole program into object code at a time. It translates one line or a single statement of a program into object code at a time.
2 It finds out the error after compiling the complete program. It finds out the error after translating a line of the program at a time.
3 The translating process is incredibly faster. The translating process is slower.
4 Compiler-based programs are difficult to code and debug. Interpreter-based programs are easier to code and debug.
5 It is usually a large program and requires larger memory space. It is usually a smaller program and requires less memory space.
6 New programming languages don't use compilers. New programming languages use interpreters.
7 For example: C, C++, Java, FORTRAN, etc. For example: BASIC, C#, PHP, etc.

Additionally, semantics refers to the meaning attached to individual words or symbols in a programming language. For example:

In QBasic:

    Print "This is programming"; P

5.7 Error (BUG)

There may be several errors inside a program that stop it from being executed. Such errors are known as bugs. These types of errors may arise from not following the proper rules of the programming language, poor problem analysis, or hardware failure. Errors in a program may result in ambiguous (not accepted/false) results. The process of removing errors from a computer program is known as debugging. There are three types of errors:

Types of Errors

Syntax Error

A syntax error arises when the programmer does not follow the proper rules or format specified by the programming language. Every programming language has its own vocabulary, punctuation, and structure, and these need to be followed meticulously. Syntax errors can be detected by language processors such as compilers and interpreters. For example, in C, every statement should end with a semicolon. If this is missed, a syntax error will occur.

Logical Error

A logical error occurs due to poorly developed logic, which hampers the solution of the problem. This type of error is not detected by language processors. For example, to calculate simple interest, we use the formula I = (P*T*R)/100. If we instead write I = P*T*R, it will result in a logical error.

Runtime Error

A runtime error occurs when a program is running or executing. These errors are not detected by the language translator. For example, if there is insufficient memory space or a peripheral device fails to turn on, a runtime error will occur.

  • Logical operators: AND, OR, NOT

5.9 Program Design Tools

There are several program design tools that are available. Some of the commonly used program design tools are algorithms, flowcharts, and pseudocode.

Algorithm

An algorithm is a step-by-step procedure or sequence of steps for solving a specific problem. Algorithms are written in simple English and are not close to any programming language.

Rules for Writing an Algorithm:

  • The algorithm should produce an output after a finite number of steps.
  • The algorithm should not depend on any programming language.
  • The algorithm should terminate after a finite number of steps.

Examples:

1. Write an algorithm to calculate simple interest.

        Step 1: Start
        Step 2: Read the value of P, T, R
        Step 3: Calculate simple interest by using I = (P * T * R) / 100
        Step 4: Display simple interest (I)
        Step 5: Stop

2. Write an algorithm to check the greatest number among two numbers.

        Step 1: Start
        Step 2: Read any two numbers a, b
        Step 3: Check if (a > b)
            If yes, Display the greatest number is "a" and go to step 4
            If no, Display the greatest number is "b" and go to step 4
        Step 4: Stop

Flowchart

A flowchart is a diagrammatic or pictorial representation of the steps involved in an algorithm to solve a particular problem. It uses several geometrical shapes and symbols for representing different actions involved in problem-solving.

There are two types of flowcharts:

  • Program Flowchart: This type of flowchart focuses on a specific problem and portrays only a single task (logic) at a time.
